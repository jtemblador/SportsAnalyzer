SLIDE 2: PROJECT OVERVIEW
================================================================================

WHAT IS THIS PROJECT?

An NFL Fantasy Football prediction system that uses machine learning to
forecast player performance for upcoming weeks.

KEY OBJECTIVES:
- Predict fantasy points, yards, touchdowns, and other stats
- Help with Daily Fantasy Sports (DFS) lineup decisions
- Achieve professional-grade accuracy (MAE < 5.0 points)

DATASET:
- 109 weeks of NFL data (2020-2025, Week 14)
- 28,000+ player-game records
- 5 positions: QB, RB, WR, TE, K
- 114 raw statistical features per game

FINAL RESULT:
✓ 4.26 MAE - EXCEEDS professional DFS platforms (4.5-5.5 MAE)
✓ Production-ready with interactive web dashboard
✓ 40 trained models delivering position-specific predictions

SLIDE 3: THE PROBLEM
================================================================================

FANTASY FOOTBALL PREDICTION CHALLENGES

1. HIGH VARIANCE
   - Quarterbacks can score 5-40 points in a single week
   - Touchdowns are random and hard to predict
   - Injuries change player roles mid-season

2. POSITION COMPLEXITY
   - QBs depend on game script (trailing teams pass more)
   - Wide receivers split targets unpredictably (3+ WRs per team)
   - Tight ends vary between blocking and receiving roles

3. VOLUME vs. EFFICIENCY
   - Fantasy scoring rewards volume, not efficiency
   - 100 rushing yards = 10 points regardless of carries
   - Opportunity matters more than talent

INDUSTRY STANDARD:
- Professional DFS platforms: 4.5-5.5 MAE
- Our goal: Beat professional standards

================================================================================
SLIDE 4: THE SOLUTION
================================================================================

MACHINE LEARNING APPROACH

ALGORITHM: CatBoost Gradient Boosting
- Handles mixed feature types (numeric, categorical)
- Resistant to overfitting
- Fast training and prediction

MODEL TYPES (3 per stat):
1. POB (Probability Over Baseline)
   - Will the player beat their average?
   - Binary classification: Yes/No

2. EVOB (Expected Value Over Baseline)
   - How much over/under their average?
   - Regression: +/- deviation

3. STAT (Direct Prediction)
   - Direct stat prediction
   - Example: 285 passing yards, 2 TDs

TOTAL MODELS: 40
- 8 models per position × 5 positions
- Position-specific hyperparameter tuning

SLIDE 5: TECHNICAL ARCHITECTURE
================================================================================

DATA PIPELINE OVERVIEW

STEP 1: RAW DATA COLLECTION
Location: data/nfl/raw/
Source: nflverse API
Format: Parquet (45MB, 90% smaller than CSV)
Columns: 114 raw statistics per game

STEP 2: FEATURE ENGINEERING
Location: data/nfl/features/[version]/
Process: Transform raw stats into predictive features
Output: 50 engineered features (V4)

STEP 3: MODEL TRAINING
Location: data/nfl/models/[version]/
Process: Train 40 CatBoost models
Output: .joblib model files

STEP 4: PREDICTION GENERATION
Location: data/nfl/predictions/[version]/
Process: Apply models to upcoming week features
Output: Predictions with confidence intervals

STEP 5: VALIDATION
Process: Compare predictions vs. actual results
Metrics: MAE (Mean Absolute Error), R² (R-squared)

SLIDE 9: KEY TECHNICAL INNOVATIONS
================================================================================

WHAT MADE V4 SUCCESSFUL?

1. VEGAS BETTING INTEGRATION (8 new features)
   - Team spread: Identifies favorites vs underdogs
   - Implied totals: Expected team scoring
   - Over/under: Game pace indicator
   - Position-specific game script indices

   WHY IT WORKS:
   Vegas oddsmakers incorporate ALL available information
   Spreads predict game flow (trailing = passing, leading = rushing)
   Totals predict overall volume opportunity

2. POSITION-SPECIFIC HYPERPARAMETERS
   - QB: depth=9, iter=500 (complex game scripts)
   - RB/WR: depth=7, iter=300 (standard complexity)
   - TE: depth=6, iter=300 (moderate complexity)
   - K: depth=3, iter=100 (simple, prevent overfitting)

   WHY IT WORKS:
   Different positions have different prediction complexity
   QBs need deeper trees to capture game script interactions
   Kickers need simple models to avoid overfitting

3. DECAY-WEIGHTED ROLLING AVERAGES (decay=0.85)
   Recent 3 games = 60.7% of total weight
   Captures recent form and trends
   Balances short-term momentum with long-term baseline

SLIDE 10: V3 FAILURE - CRITICAL LEARNING
================================================================================

WHY DID V3 FAIL? (EPA & Efficiency Metrics)

WHAT WE ADDED:
- EPA (Expected Points Added): passing_epa, rushing_epa, receiving_epa
- Efficiency metrics: CPOE, PACR, RACR, WOPR
- Advanced analytics: YAC, air yards share, target distribution
- Position-specific decay factors
- 15 new features (42 → 57 columns)

RESULT: 4.66 MAE (identical to V2, ZERO improvement)
TRAINING TIME: 3.6 hours (vs V2's 25 minutes)

WHY IT FAILED:
1. EPA is DERIVED from existing statistics
   - Already captured by V2's volume features
   - High correlation = no new information

2. Fantasy scoring rewards VOLUME, not EFFICIENCY
   - 100 yards on 25 carries = 10 points
   - 100 yards on 18 carries = 10 points (same score!)
   - Efficiency metrics don't predict opportunity

3. OVERFITTING RISK
   - 57 features on 28,000 records
   - Too many features for dataset size
   - Complexity hurt generalization

CRITICAL LESSON:
Domain knowledge can mislead. Simple features that predict context and
opportunity beat complex analytics that measure efficiency.

V3 was rejected but valuable - taught us what doesn't work.

SLIDE 14: KEY LEARNINGS & INSIGHTS
================================================================================

TECHNICAL LESSONS

1. SIMPLE FEATURES BEAT COMPLEX ANALYTICS
   V2's variance/trends (42 features) > V3's EPA (57 features)
   Lesson: Feature quality > feature quantity

2. CONTEXT BEATS PLAYER METRICS
   V4's Vegas lines > V3's efficiency stats
   Lesson: Predict opportunity, not talent

3. POSITION-SPECIFIC TUNING MATTERS
   QB needs depth=9, K needs depth=3
   Lesson: One size does NOT fit all

4. ITERATION AND VALIDATION ARE CRITICAL
   V3 seemed promising but testing revealed zero improvement
   Lesson: Empirical validation > theoretical soundness

5. DOCUMENTATION PREVENTS REWORK
   Version-controlled features, models, predictions
   Lesson: Reproducibility is essential

DOMAIN INSIGHTS

1. FANTASY FOOTBALL REWARDS VOLUME, NOT EFFICIENCY
   100 yards = 10 points regardless of efficiency
   EPA and advanced analytics don't predict fantasy production

2. GAME SCRIPT DRIVES QUARTERBACK PERFORMANCE
   Trailing teams pass 2x more than leading teams
   Vegas spreads and totals capture game flow

3. TIGHT ENDS ARE MOST PREDICTABLE (WITH RIGHT FEATURES)
   V2 variance distinguished elite vs blocking TEs
   V4 Vegas context predicted game script effects
   62% improvement: worst position → best position

4. WIDE RECEIVERS ARE HARDEST TO PREDICT
   3+ WRs per team splitting targets unpredictably
   Defensive schemes (shadow coverage) vary game-to-game
   Needs target distribution model, not just team totals

5. RUNNING BACKS ARE NATURALLY PREDICTABLE
   Lead backs get consistent 20-25 touches
   Role clarity makes prediction easier
   Already achieved 4.54 MAE in V1

- Position-specific optimization

SLIDE 18: CHALLENGES OVERCOME
================================================================================

MAJOR TECHNICAL CHALLENGES

1. QUARTERBACK VOLATILITY (V1-V3: 6.56-7.26 MAE)
   Problem: Game script-dependent performance
   Solution: Vegas betting lines capture game flow
   Result: V4 achieved 4.67 MAE (35% improvement)

2. TIGHT END PREDICTION (V1: 6.15 MAE, worst position)
   Problem: Mixed role (blocker vs receiver)
   Solution: Variance features + Vegas game context
   Result: V4 achieved 2.34 MAE (62% improvement, best position)

3. FEATURE SELECTION (V3 had 57 features, zero improvement)
   Problem: Complex features caused overfitting
   Solution: Focus on context over analytics
   Result: V4 with 50 features achieved 4.26 MAE

4. VALIDATION METHODOLOGY
   Problem: Need fair comparison across versions
   Solution: Held-out test set (last 3 weeks)
   Result: Prevented overfitting, revealed V3 failure

5. DATA EFFICIENCY
   Problem: CSV files were 450MB+ and slow to load
   Solution: Parquet columnar format
   Result: 45MB storage (90% reduction), 10x faster reads

6. VERSION CONTROL
   Problem: Risk of overwriting previous work
   Solution: Version-controlled folders (v1/, v2/, v3/, v4/)
   Result: Easy rollback, version comparison, reproducibility

7. HYPERPARAMETER TUNING
   Problem: One-size-fits-all parameters suboptimal
   Solution: Position-specific tuning (QB depth=9, K depth=3)
   Result: Improved accuracy across all positions

8. PREDICTION CONFIDENCE
   Problem: Point predictions don't show uncertainty
   Solution: Confidence intervals (lower/upper bounds)
   Result: Risk-aware predictions for DFS strategy

seemed promising but failed in practice. Simple features that predict context
and opportunity (variance, trends, Vegas lines) outperformed complex player
analytics. Always validate empirically.

DOCUMENTATION
================================================================================

Created DATA_FOLDER_STRUCTURE.md with complete breakdown:

1. RAW Data (data/nfl/raw/)
   - 103 files (player_stats_{season}_week_{week}.parquet)
   - 114 columns: actual game stats from nflverse
   - Example: fantasy_points_ppr, passing_yards, receiving_yards
   - Size: ~150KB per file, 15MB total
   - Usage: Source for feature engineering & training targets

2. FEATURES Data (data/nfl/features/{version}/)
   - 102 files per version (features_{season}_week_{week}.parquet)
   - V1: 33 columns (rolling avgs, opponent rank, basic trends)
   - V2: 42 columns (all v1 + variance + trend features)
   - Size: ~50KB per file, 5MB total per version
   - Usage: Input features for ML model training

3. MODELS (data/nfl/models/{version}/)
   - 40 models per version (*.joblib files)
   - Types: *_evob (regression), *_pob (classification)
   - Size: ~2-4MB per model, 91MB total per version
   - Usage: Generate predictions for weeks

4. PREDICTIONS (data/nfl/predictions/{version}/)
   - Variable number of files (predictions_{season}_week_{week}.parquet)
   - 15 columns: predicted values, probabilities, confidence intervals
   - Size: ~100KB per file
   - Usage: Displayed in dashboard, validated against actuals

================================================================================
BENEFITS OF VERSIONING
================================================================================

1. Data Preservation
   - V1 features safe and intact
   - Can recreate v1 models anytime
   - No data loss during experiments

2. Schema Consistency
   - Each version has consistent feature schema
   - No mixed old/new column issues
   - Clean training data for each version

3. Comparison Capability
   - Can train v1 models on v1 features
   - Can train v2 models on v2 features
   - Fair apple-to-apples comparison

4. Rollback Safety
   - If v2 doesn't improve, keep using v1
   - If v3 experiment fails, rollback to v2
   - No need to regenerate from scratch

5. Reproducibility
   - Each version is self-contained
   - Can reproduce any experiment
   - Clear lineage: features → models → predictions

6. Experimentation Freedom
   - Try v3 with different decay factor
   - Try v4 with weather data
   - Try v5 with ensemble features
   - All without destroying previous work

FEATURES VS PREDICTIONS CONFUSION (RESOLVED)
--------------------------------------------
User Question: "Are features and predictions redundant?"

ANSWER: NO - Completely different!

FEATURES (Input to models):
  Location: data/nfl/features/v2_variance_trends_mae4.66/
  Rows: 337 players per week
  Columns: 42 feature columns
  Contains: rolling_avg_fantasy_pts, passing_yds_variance, opponent_defense_rank, etc.
  Purpose: Engineered input data for ML models

PREDICTIONS (Output from models):
  Location: data/nfl/predictions/v2_variance_trends_mae4.66/
  Rows: 1489 predictions per week (multiple stats per player)
  Columns: 15 prediction columns
  Contains: predicted_value, confidence_lower, confidence_upper, probability_over, etc.
  Purpose: Forecasted performance for upcoming games

Analogy: Features = Ingredients, Predictions = Finished Meal

V1 VS V2 FEATURES DIFFERENCES (RESOLVED)
-----------------------------------------
User Question: "Why can't we reuse V1 features for V2 models?"

ANSWER: Features are CALCULATED differently between versions

V1 Features (34 columns):
  - Decay factor: 0.9 (10% decay per game)
  - No variance features
  - No trend features
  - Basic rolling averages only

V2 Features (42 columns):
  - Decay factor: 0.85 (15% decay - more recent emphasis)
  - NEW: 8 variance features (boom/bust indicators)
  - NEW: 8 trend features (hot/cold streaks)
  - All V1 features + improvements

Example values for A.Rodgers Week 10:
  V1 rolling_avg_fantasy_pts: 15.69
  V2 rolling_avg_fantasy_pts: 15.72 (different due to decay)
  V2 fantasy_pts_variance: 5.3 (NEW - doesn't exist in V1)
  V2 fantasy_pts_trend: +2.1% (NEW - doesn't exist in V1)

Conclusion: Each model version needs its own feature version

QB-Specific Feature Additions:
  1. Weather Features:
     - wind_speed (QBs struggle in wind)
     - temperature (cold affects accuracy)
     - is_dome_game (eliminates weather)

  2. Rest & Schedule Features:
     - days_rest (Thursday games hurt QBs)

     - red_zone_efficiency (TD opportunities)
     - deep_ball_rate (big play potential)
     - game_script_tendency (run-heavy vs pass-heavy)
     - week_of_season (early season rust, playoff push)

     - QB: 0.90 (need MORE history for game planning)
     - RB: 0.85 (recent form matters)
     - WR: 0.85 (recent form matters)

  - QBs need game planning context (more history)
  - Missing critical situational features (weather, rest)

WHAT THE USER LEARNED
================================================================================

1. Features ≠ Predictions
   - Features: Input to models (engineered stats)
   - Predictions: Output from models (forecasts)
   - NOT redundant, completely different data

2. V1 and V2 features are incompatible
   - Different number of columns (34 vs 42)
   - Different decay factors (0.9 vs 0.85)
   - Different calculation methods
   - V2 models trained on 42 features, can't use 34 feature input

3. File versioning strategy
   - feature_engineer.py = Current version (V2)
   - ml_models.py = Version-agnostic (works with any version)
   - train_models.py = Version-agnostic (CLI accepts version)
   - v2_retrain.py = Version-specific workflow

4. How to improve QB predictions
   - Priority 1: Add QB-specific features (weather, pressure, rest)
   - Priority 2: Position-specific model hyperparameters
   - Priority 3: Position-specific decay factors
   - ml_models.py CAN be optimized (hyperparameters)
   - train_models.py does NOT need optimization (just a wrapper)

5. Data pipeline flow
   Raw → Features → Models → Predictions → Validation
   Each step creates different data in versioned folders

Task 0.1 — Schedule Fetcher (Vegas Lines + Weather + Game Context)
Date: 2026-04-10
Status: Complete
Tests: 18 new (35 total across project)

Goal
-----
Pull NFL schedule data from nflreadpy for seasons 2018-2025 to get free historical
Vegas lines, weather, and game context. This replaces the paid Odds API approach
entirely and gives us 100% coverage of spreads, totals, and moneylines.

Approach
--------
- Used nfl.load_schedules() which returns 46 columns per game including spread_line,
  total_line, moneylines, temp, wind, roof, surface, rest days, stadium, coaches, etc.
- One Parquet file per season: data/nfl/schedules/schedules_{season}.parquet
- Added 2 derived columns: home_implied_total and away_implied_total calculated from
  spread_line and total_line. Formula: home = (total + spread) / 2, away = (total - spread) / 2.
  Spread convention: negative = home team favored.
- Built ScheduleFetcher as a standalone class registered in NFLDataPipeline, establishing
  the pattern all future fetchers (Tasks 0.2-0.12) will follow.
- Season range decision: pull 2018-2025 (8 seasons). 2018-2019 serve as warm-up data
  for rolling averages. Models train on 2020-2025 only.

Implementation
--------------
Created:
- src/nfl/data/fetch_schedules.py — ScheduleFetcher class (fetch_season, fetch_all, load_season)
- data/nfl/schedules/ — 8 Parquet files (2018-2025), 2,227 total games
- tests/test_fetch_schedules.py — 18 tests across 3 test classes

Modified:
- src/nfl/data/pipeline.py — added ScheduleFetcher registration, fetch_all_player_stats(),
  fetch_all() orchestrator, cleaned up duplicate docstring and unused imports, updated
  file path in module docstring
- data/nfl/raw/ — backfilled 2018-2019 player stats (34 weeks) + 2025 weeks 15-18

Validation
----------
- Spot-check: 2024 Week 1 BAL@KC — spread 3.0, total 46.0, home_implied 24.5,
  away_implied 21.5, home_score 27, away_score 20. All correct.
- Implied totals verified: home_implied + away_implied = total_line for all 285 games in 2024.
- Spread line: 0 nulls across all 8 seasons (100% coverage).
- Weather: temp/wind ~64% populated (dome/closed roof games have no outdoor weather — expected).
  Roof and surface are 100% populated.
- Rest days: all non-null, range 3-14 days (reasonable).
- 2018-2019 are 16-game seasons (weeks 1-17+ playoffs) — handled correctly.
- Pipeline fetch_all() correctly orchestrates both player stats and schedules, skips existing data.

Lessons Learned
---------------
- nflreadpy provides free historical Vegas lines via load_schedules() — no paid API needed.
  This was a major discovery that changes the project architecture (paid Odds API can be removed).
- Weather data is only ~64% complete because dome/closed roof games have no outdoor
  temp/wind readings. When building features, fill with neutral defaults (72°F, 0 mph)
  rather than filling in the raw data.
- 2018-2019 have different season lengths (17 weeks max vs 18+ for 2021+). The fetcher
  handles this naturally since load_schedules() returns whatever exists.
- Existing raw player_stats were missing 2018-2019 entirely. Backfilled 34 weeks.
  Also picked up 2025 weeks 15-18 which had been played since last fetch.
- The unified pipeline pattern (fetcher class + register in pipeline) works well.
  Future tasks should follow the same pattern.

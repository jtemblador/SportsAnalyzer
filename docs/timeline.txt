PROJECT OVERVIEW
================================================================================

The system predicts NFL player fantasy performance using machine learning models
trained on 6 seasons (2020-2025) of weekly player statistics. It uses three
complementary prediction approaches:

1. POB (Probability Over Baseline): Binary classifier predicting if player
   will exceed their rolling average
2. EVOB (Expected Value Over Baseline): Regression predicting point differential
   from rolling average
3. Stat Predictor: Specialized models for individual stats (yards, TDs, etc.)

PHASE 1: FEATURE ENGINEERING - ✅ COMPLETE
================================================================================

File: src/nfl/feature_engineer.py
Status: COMPLETE - 108 feature files generated (2020-2025, weeks 1-18)

What it does:
- Loads raw player stats from data/nfl/raw/
- Calculates rolling averages (6-game window with 0.9 decay)
- Calculates opponent adjustments based on defense rankings
- Calculates usage trends (carry share, target share, etc.)
- Generates 20-30 features per position
- Saves engineered features to data/nfl/cleaned/

Output: 108 parquet files in data/nfl/cleaned/
Example: features_2025_week_5.parquet

Feature Types:
- Universal: rolling_avg_fantasy_pts, rolling_avg_fantasy_ppr,
  opponent_defense_rank, games_in_history, has_sufficient_data
- QB-specific: rolling_avg_passing_yds/tds/interceptions,
  opponent_pass_defense_rank, pass_attempts_trend
- RB-specific: rolling_avg_rushing_yds/tds/carries/receptions,
  opponent_rush_defense_rank, carry_share_trend
- WR/TE-specific: rolling_avg_receiving_yds/tds/receptions/targets,
  target_share_trend, air_yards_share_trend
- K-specific: rolling_avg_fg_made/att, rolling_avg_pat_made

PHASE 2: ML MODEL CLASSES - ✅ COMPLETE
================================================================================

File: src/nfl/ml_models.py
Status: COMPLETE - All model classes implemented and tested

What it contains:

1. POBModel (Probability Over Baseline) - Lines 30-253
   - Binary classification model
   - Predicts probability (0-1) that player exceeds rolling average
   - Uses ensemble of 4 models: XGBoost, LightGBM, CatBoost, RandomForest
   - Methods: train(), predict(), evaluate(), save_model(), load_model()
   - Only trained for fantasy_points_ppr stat

2. EVOBModel (Expected Value Over Baseline) - Lines 255-510
   - Regression model
   - Predicts point differential from rolling average
   - Uses ensemble of 4 models (same as POB)
   - Includes predict_with_intervals() for confidence ranges
   - Trained for all stats (fantasy points, yards, TDs, etc.)

3. StatPredictor (Individual Stat Predictor) - Lines 513-635
   - Specialized predictor for specific stats
   - Chooses appropriate model type based on stat distribution
   - Count data (TDs, INTs): Standard regression with non-negative constraint
   - Continuous data (yards): Standard regression
   - Percentage data: Logistic regression

4. NFLModelPipeline (Main Orchestrator) - Lines 638-1072
   - Loads features and targets from parquet files
   - Splits data: 70% train, 15% validation, 15% test
   - Trains all models for all positions
   - Generates predictions for any week
   - Saves models and predictions to disk

WHAT TO WATCH OUT FOR
================================================================================

Common Issues and Solutions:

1. MEMORY ERRORS during training/prediction
   Symptom: "MemoryError" or system freezes
   Solution: Process one position at a time, or use fewer ensemble models

2. MISSING FEATURES warning
   Symptom: "⚠️  Missing N features" messages
   Solution: Expected for some players - feature_engineer may not calculate
             all features for all positions. Models handle this gracefully.

3. PREDICTION FILE IS EMPTY
   Symptom: predictions_{season}_week_{week}.parquet has 0 rows
   Solution: Check if features_{season}_week_{week}.parquet exists
             Week 1 of each season may have insufficient history

4. PREDICTIONS ARE ALL IDENTICAL
   Symptom: All players have same predicted value
   Solution: Features may not be loading correctly. Check feature_engineer
             output for that week.

5. CONFIDENCE INTERVALS TOO WIDE
   Symptom: Intervals like [5.0, 45.0] for 15-point prediction
   Solution: Model ensemble has high variance - may need hyperparameter tuning
             or more training data

6. POB PROBABILITY ALWAYS ~50%
   Symptom: All POB predictions between 48-52%
   Solution: Model can't distinguish - features may be too similar.
             This is actually OK - means players are performing near baseline.

================================================================================

Training Data Split:
- Train: 70% (2020-2023 all weeks)
- Validation: 15% (2024 weeks 1-9)
- Test: 15% (2024 weeks 10-18 + 2025 weeks 1-13)

Ensemble Approach:
Each model type uses 4 algorithms and averages their predictions:
1. XGBoost (200-300 iterations)
2. LightGBM (200-300 iterations)
3. CatBoost (200 iterations)
4. Random Forest (200 estimators)

Feature Scaling:
- StandardScaler applied to all features before training
- Scaler saved with each model for consistent prediction scaling

Validation Metrics:
- POB (Classification): Accuracy, Precision, Recall, F1-Score
- EVOB (Regression): MAE, RMSE, R²
- Stat Predictor: MAE, RMSE, R²


================================================================================
PREDICTION WORKFLOW
================================================================================

For a player in Week N:

1. Load raw stats for Weeks N-6 through N-1 (history)
2. Engineer features using FeatureEngineer
3. Load appropriate position-specific models
4. Make 3 predictions:

   a) POB Prediction (Classification)
      - Input: Engineered features
      - Output: Probability (0-1) of beating rolling average
      - Example: 0.68 = 68% chance to beat baseline

   b) EVOB Prediction (Regression)
      - Input: Engineered features
      - Output: Point differential from rolling average
      - Example: +2.4 points above baseline
      - Includes confidence interval: [+0.5, +4.3]

   c) Stat Prediction (Regression)
      - Input: Engineered features
      - Output: Actual projected value for stat
      - Example: 287.5 passing yards

5. Combine predictions:
   Projected Score = Rolling Average + EVOB Prediction
   Example: 24.3 (baseline) + 2.4 (EVOB) = 26.7 points

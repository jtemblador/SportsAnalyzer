# Project Context Summary for AI Assistant

## **Project Overview**
Building a **multi-sport analytics platform** (currently focused on NFL) that predicts player performance using machine learning. This is a senior design project with a focus on local-first architecture, transparent predictions, and explainable AI.

---

## **Current Project Status**

### **✅ Phase 1: COMPLETE (Data Infrastructure)**
- Fully functional ETL pipeline using `nflreadpy` library
- Automated incremental data fetching (only downloads new weeks)
- Local storage: 108+ parquet files covering 2020-2025 NFL seasons
- Streamlit dashboard with interactive filtering and visualizations
- Data stored in `data/nfl/raw/` with naming: `player_stats_{SEASON}_week_{WEEK}.parquet`

### **⏳ Phase 2: IN PROGRESS (Feature Engineering & ML Models)**
- About to implement feature engineering (rolling averages, opponent adjustments, usage metrics)
- Planning to build POB (Probability Over Baseline) and EVOB (Expected Value Over Baseline) models
- Will use XGBoost/LightGBM for predictions
- Training on 2020-2025 historical data

### **📋 Phase 3: PLANNED (Explainable AI)**
- SHAP values for feature importance
- GPT integration for natural language explanations

## **Key Design Principles**

1. **Local-First:** No cloud dependencies, all data stored locally
2. **Transparent:** Raw data never modified, stored exactly as received from API
3. **Incremental:** Only fetch missing weeks, never re-download
4. **Reproducible:** Same data + same code = same results
5. **Explainable:** Show WHY predictions are made, not just the predictions

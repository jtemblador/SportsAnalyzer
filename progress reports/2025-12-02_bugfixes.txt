================================================================================
TECHNICAL NOTES
================================================================================

Why Variance Features are Optional:
- Variance requires 2+ games of history (calculate_variance line 108)
- Trends require 4+ games (3 recent + 1 older)
- New players or early season have limited history
- Filling with 0 is safe: means "no boom/bust tendency known"
- Models learned to interpret 0 as neutral during training

Why Full Regeneration is Needed:
- Feature schema changed: 9 features → 13 features (for WR)
- Old files: no variance columns
- New files: have variance columns
- Mixed data causes column mismatch during training
- Must regenerate ALL to ensure consistency

Why 50 Sample Threshold:
- ML models need minimum data to learn patterns
- 50 is reasonable for gradient boosting models
- QB/RB/TE should have 1000+ samples after full regen
- Old issue was mixed schemas, not actual data shortage

================================================================================
KEY INSIGHTS
================================================================================

1. Feature Schema Versioning Matters:
   - Can't mix old/new feature schemas
   - Must regenerate ALL when schema changes
   - Should have caught this in design phase

2. Off-by-One Errors are Sneaky:
   - Week numbering inconsistency caused by old "predict next week" logic
   - Important to audit all week references
   - Test files helped catch this quickly

3. Graceful Degradation is Good:
   - Auto-filling missing features is better than crashing
   - Models can still make predictions with partial info
   - User experience much better (warnings vs errors)

4. Test Before Full Training:
   - Quick test script saved 20 minutes of waiting
   - Verified fixes work before expensive operations
   - Good practice for ML pipelines

================================================================================
LESSONS FOR FUTURE
================================================================================

1. When changing feature schema:
   - Always regenerate ALL data
   - Don't rely on "skip if exists" logic
   - Add version tracking to feature files

2. When predicting on new data:
   - Expect missing features (new players, early season)
   - Fill with sensible defaults (0, mean, median)
   - Log what's being filled for debugging

3. When modifying week logic:
   - Grep for ALL week references
   - Check both data structures AND filenames
   - Test with specific week numbers

4. When training takes 20+ minutes:
   - Create quick tests to verify fixes
   - Fail fast before expensive operations
   - Save user time and frustration

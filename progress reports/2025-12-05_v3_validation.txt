5. ROOT CAUSE ANALYSIS - WHY V3 FAILED

   V3 Added 15 New Features:
   - passing_epa, rushing_epa, receiving_epa
   - cpoe (Completion % Over Expected)
   - pacr, racr (Receiving efficiency)
   - wopr (Weighted Opportunity Rating)
   - target_share, air_yards_share
   - passing_yac, receiving_yac
   - fg_made_40_49, fg_made_50_plus
   - Position-specific decay (0.80-0.90 vs uniform 0.85)

   Why These Didn't Help:

   A) FEATURE REDUNDANCY
      - EPA is calculated from yards/TDs already in V2 rolling averages
      - CPOE correlates with existing passing_yards variance
      - Models already learned these patterns from simpler features
      - Adding derived features doesn't add new information

   B) EFFICIENCY вү  FANTASY VOLUME
      - PACR/RACR measure yards per target (efficiency)
      - Fantasy points need total targets (volume)
      - A WR can be 100% efficient on 2 targets = 4 points
      - A WR can be 50% efficient on 10 targets = 15 points
      - V3 optimized for wrong metric

   C) POSITION DECAY INEFFECTIVE
      - QB decay 0.90 vs 0.85: No difference in MAE
      - TE decay 0.80 vs 0.85: Marginal 1.4% improvement
      - V2's variance features already capture position volatility
      - Uniform decay is sufficient

   D) OVERFITTING ON SMALL DATASETS
      - QB: only 2,893 training records
      - K: only 2,592 training records
      - 57 features on limited data вҶ’ models memorize vs generalize
      - V2's 42 features are already at capacity

6. WHY V2 SUCCEEDED (Validated by V3 Failure)

   V2's Winning Features:
   - Variance features: Capture boom/bust tendency
   - Trend features: Identify hot/cold streaks
   - Decay 0.85: Optimal recent-game weighting
   - 42 columns: Sweet spot for 28K training records

   Key Insight: Simpler features generalize better
   - Rolling averages implicitly encode efficiency
   - Variance captures consistency (volume + efficiency combined)
   - Trends identify role changes and usage patterns
   - Don't need explicit EPA when model learns from outcomes

================================================================================
LESSONS LEARNED
================================================================================

1. SIMPLE BEATS COMPLEX
   - V2's variance/trend features outperform V3's EPA analytics
   - Models find patterns in simple features automatically
   - More features вү  better predictions

2. DOMAIN KNOWLEDGE CAN MISLEAD
   - EPA is "better" than raw stats for evaluating player skill
   - But skill evaluation вү  fantasy prediction
   - Fantasy = volume Г— opportunity Г— efficiency
   - V2's simple features capture all three implicitly

3. POSITION-SPECIFIC TUNING DOESN'T HELP
   - Different decay factors (0.80-0.90) made no difference
   - Variance features already capture position characteristics
   - One-size-fits-all (0.85) is sufficient

4. FEATURE ENGINEERING HAS DIMINISHING RETURNS
   - V1 вҶ’ V2: Added 8 features, gained 9% accuracy
   - V2 вҶ’ V3: Added 15 features, gained 0% accuracy
   - We've likely hit the ceiling for player-level features

5. VALIDATION IS CRITICAL
   - Without compare_versions.py fix, we'd think V3 improved
   - Automated validation catches regressions
   - Never trust training metrics, always validate on test set

6. DOCUMENT FAILURES
   - V3 "failure" taught us V2's strengths

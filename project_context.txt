PROJECT OVERVIEW
================================================================================

WHAT IS THIS PROJECT?
---------------------
NFL Fantasy Football Prediction System using Machine Learning.
Predicts player performance (fantasy points, yards, TDs, etc.) for upcoming weeks
to help with DFS (Daily Fantasy Sports) lineup decisions.

GOAL:
-----
Predict NFL player fantasy performance with Mean Absolute Error (MAE) < 5.0 points.
Current best: V4 with 4.26 MAE (EXCEEDS professional DFS range: 4.5-5.5)

CURRENT STATUS:
---------------
✓ V4 PRODUCTION (MAE: 4.26) - 17% better than V1, exceeds industry standards
✓ V2 backup model (MAE: 4.66) - reliable fallback
✓ V3 rejected (MAE: 4.66) - EPA/efficiency metrics didn't help
✓ 109 weeks of historical data (2020-2025 Week 14)
✓ 40 trained V4 models (8 per position: QB, RB, WR, TE, K)
✓ Web API ready for V4 integration (app.py)

================================================================================
DATA PIPELINE FLOW
================================================================================

RAW DATA → FEATURES → MODELS → PREDICTIONS → VALIDATION
   ↓          ↓          ↓          ↓            ↓
Step 1     Step 2     Step 3     Step 4      Step 5

STEP 1: RAW DATA COLLECTION
----------------------------
Location: data/nfl/raw/
Files: player_stats_YYYY_week_W.parquet
Columns: 114 (all NFL stats from API)
Source: NFL stats API (scraped weekly)

Example:
  player_stats_2025_week_13.parquet
  Contains: player_name, team, position, passing_yards, rushing_yards, etc.

STEP 2: FEATURE ENGINEERING
----------------------------
Input: Raw stats from Step 1
Output: Engineered features (rolling averages, trends, variance)
Location: data/nfl/features/[VERSION]/

Files Created:
  features_YYYY_week_W.parquet

Example transformation:
  Raw: P.Mahomes had 250, 300, 275 yards in last 3 games
  Feature: rolling_avg_passing_yds = 275 (with decay weighting)
            passing_yds_variance = 25.0 (consistency indicator)
            passing_yds_trend = +5.2% (recent improvement)

V1 Features: 34 columns, decay=0.9
V2 Features: 42 columns, decay=0.85 (+ variance & trend features)
V3 Features: 57 columns, position-specific decay (+ EPA, efficiency metrics)
V4 Features: 50 columns (V2's 42 + 8 Vegas odds features)

STEP 3: MODEL TRAINING
-----------------------
Input: Features from Step 2
Output: Trained ML models
Location: data/nfl/models/[VERSION]/

Models per position: 8 models
  - fantasy_points_ppr_pob.joblib (Probability Over Baseline)
  - fantasy_points_ppr_evob.joblib (Expected Value Over Baseline)
  - fantasy_points_ppr_stat.joblib (Direct stat prediction)
  - passing_yards_evob.joblib
  - passing_yards_stat.joblib
  - passing_tds_evob.joblib
  - passing_tds_stat.joblib
  - passing_interceptions_evob.joblib

Total: 40 models across 5 positions (QB, RB, WR, TE, K)

Model Types:
  POB (Probability Over Baseline): Binary classification - will they beat their average?
  EVOB (Expected Value Over Baseline): How much over/under their average?
  STAT: Direct stat prediction (yards, TDs, etc.)

STEP 4: PREDICTION GENERATION
------------------------------
Input: Features + Trained Models
Output: Predictions for upcoming week
Location: data/nfl/predictions/[VERSION]/

Files Created:
  predictions_YYYY_week_W.parquet

Contains:
  player_name, position, stat, predicted_value, confidence_lower,
  confidence_upper, probability_over, baseline

Example:
  P.Mahomes, QB, passing_yards, 285.5, [265-305], 0.62, 275
  (Predicting 285 yards, 62% chance to beat his 275 baseline)

STEP 5: VALIDATION
------------------
Input: Predictions + Actual Results
Output: Accuracy metrics (MAE, R², by position)
Location: Testing scripts

Compares predicted vs actual to calculate:
  - MAE (Mean Absolute Error): Average points off
  - R² (R-squared): How well predictions explain variance
  - Position-specific accuracy

HOW TO USE THIS PROJECT

SCENARIO 1: TRAIN A NEW VERSION (V3)
-------------------------------------
1. Create new feature engineer:
   cp src/nfl/feature_engineer.py src/nfl/v3_feature_engineer.py
   (Edit v3_feature_engineer.py to add improvements)

2. Create workflow script:
   cp src/nfl/v2_retrain.py src/nfl/v3_retrain.py

   python src/nfl/v3_retrain.py

4. Result:
   data/nfl/features/v3_*/
   data/nfl/models/v3_*/
   data/nfl/predictions/v3_*/

You trained models but only have predictions for 3 weeks.

python testing/validate_accuracy.py v2_variance_trends_mae4.66 10 11 12

-----------------------------
python testing/compare_versions.py v1_baseline_mae5.14 v2_variance_trends_mae4.66 10 11 12

================================================================================
KEY METRICS EXPLAINED
================================================================================

MAE (Mean Absolute Error)
--------------------------
Average points off from actual performance.
Lower is better.

Example: Predicted 18.5 points, actual 15.0 points → Error: 3.5
         Predicted 22.0 points, actual 25.0 points → Error: 3.0
         MAE = (3.5 + 3.0) / 2 = 3.25

Professional DFS MAE Range: 4.5 - 5.5
Our V4 MAE: 4.26 ✓✓ (EXCEEDS pro range!)

R² (R-squared)
--------------
How well predictions explain actual variance.
Range: 0.0 (random) to 1.0 (perfect)

0.0 - 0.2: Poor
0.2 - 0.4: Fair
0.4 - 0.6: Good
0.6 - 0.8: Very Good
0.8 - 1.0: Excellent

Our V4 R²: 0.442 (Good) ← Improved predictive power
